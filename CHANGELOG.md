# Changelog

All notable changes to SpeedKalandra are tracked here. The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/), and the project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

Each release section is short and user-facing; engineering rationale lives next to the code (or in `ARCHITECTURE.md` when it shapes the design). Pre-release `v17.x` tags are listed at the end of this file as historical context only; they are no longer kept in source comments.

## [Unreleased]

### Added

- **`.gitattributes` fixing line endings and binary handling.** `*.ahk`/`*.ps1`/`*.bat`/`*.cmd` pinned to CRLF (Windows-native tooling — the AHK error dialog and Notepad++ expect it; the build/tray flows only run on Windows); `*.md`/`*.csv`/`*.yml`/`*.yaml`/`*.json` pinned to LF (the cross-platform default, also what GitHub renders cleanest); `*.ini` and images/archives marked binary so git never tries to text-normalize a UTF-16 LE BOM file that `IniWrite` produced (no INI is committed today, but the rule is a guardrail for any future template/fixture). Existing files were intentionally **not** mass-renormalized (`git add --renormalize .` was not run) — only new commits respect the new rules. A separate PR can renormalize the whole repo at once if desired; the diff would be massive and worth handling on its own, not bundled here.

### Fixed

- **Zone internal IDs leaking as human names (engine `G1_2` shown/saved instead of `Clearfell`).** PoE2's `Client.txt` emits `[SCENE] Set Source [<raw>]` with `<raw>` being either a human zone name ("Mud Burrow") or the engine's internal id ("G1_3"). The old `LogMonitorService` copied the raw text straight into `Events.ZoneChanged{zoneName: raw, sceneId: raw}`, so when the engine emitted the id, every downstream consumer keyed its data by the id: `ZoneTrackingService._totals["G1_3"]`, `RunStatsPlotBuilder._AddZoneDetails` finding `FindByName("G1_3")` empty (fell back to category="mapa", act=0, label="G1_3"), `RunHistoryRepository.Save` writing `[totals] G1_3=ms` to `data/runs/{runId}.ini`, and `PersonalBestService._zonePbs["G1_3"]` in `personal_bests.ini`. `GetActTotals()` silently dropped the zone from per-act tallies because `FindByName` of the id failed. The fix resolves the raw text to the canonical human name from `ZonesCatalog` at the single publisher (`LogMonitorService._ResolveZoneToHumanName`): try `FindByName` first (catches human-name lines, recovers canonical case), then `FindById` (catches engine-id lines, returns the catalog's display name), then pass through the raw text for unknown zones (a new game patch or randomized instance still surfaces in the overlay/history — just without act/isTown metadata). `Events.ZoneChanged.sceneId` continues to carry the raw text for diagnostics. The catalog is an optional constructor parameter on `LogMonitorService`; tests/headless setups without one preserve the pre-fix behaviour. The legacy `"You have entered <zone>"` parse branch is also routed through the resolver — not observed in current PoE2 `Client.txt` (every real-world `ZoneChanged` fires from `[SCENE]` today), retained defensively in case the engine reintroduces the line. 8 new tests across `log_monitor_service_tests.ahk` (resolution by id, resolution by name with case recovery, fallback for unknown zones, no-catalog passthrough, constructor type-check) plus 1 end-to-end guardrail in `speedkalandra_app_integration_tests.ahk` that proves `[SCENE] Set Source [G1_3]` lands in `zoneTracker._totals["Mud Burrow"]` (not `["G1_3"]`). Historical run files on disk that were saved with raw-id keys keep working (Load is unchanged); the user can `Reset PBs` or import-rebuild to regenerate clean PB keys from new runs.

- **INI-breaking characters in textual fields now rejected at import and on save.** Run files (`data/runs/{runId}.ini`) and the personal-bests INI rely on `\r`, `\n`, `[` and `]` having reserved structural meaning. Before this fix nothing stopped a hand-edited or maliciously crafted import payload from putting any of those characters into `profile`, `patch`, `firstTs`, `runId`, a zone-name key under `totals`, a detail row's `category`/`label`/`note`/`timestamp`, the PB `runPbRunId`, or a `zonePbs` zone key. A single `\n` would silently split a row across two INI lines on the next save, making the `[details].count` value mismatch the actual rows and corrupting `Load`. The fix is two layers: (1) `RunExportFormat.ValidateSchema` now scans every textual field on every run plus the PB block, and reports `"runs[N].fieldName contains INI-breaking character (\\n)"` so the user sees which field needs cleaning before the import touches disk; (2) `RunHistoryRepository._SerializeBuildResultToIni` defensively throws `ValueError` on the same character set, routed through the existing `WarningSink` so `Save` returns `false` and emits a warning identifying the failed `runId` — protection against a future parser regression smuggling bad text in through some other code path. Safe punctuation (`=`, `;`, `|`, parens, long strings, Unicode) is explicitly preserved by a round-trip test. 17 new tests: 9 covering each rejected field/character combination at the import layer (`run_export_format_tests.ahk`), 5 round-trip tests confirming safe characters survive Save/Load unchanged, and 3 defensive-throw tests confirming a bad field surfaces through the `WarningSink` instead of corrupting the file (`run_history_repository_tests.ahk`).

- **Removed accidental Exit hotkey (`Ctrl+Alt+Q`).** The entry script registered `^!q::ExitApp()` as a global hotkey. On ABNT2 and other international keyboards `AltGr+Q` is reported by Windows as `LCtrl+RAlt+Q`, which AHK's `^!` matches exactly — pressing AltGr+Q during a run closed the app and lost in-progress state. The tray menu "Exit" item already covers the use case, so the global hotkey was redundant and unsafe. Removed entirely; tray-menu Exit is the documented way to close the app.

- **`RunHistoryRepository.Save` is now atomic.** The previous implementation made roughly N direct `IniWrite` calls against `data/runs/{runId}.ini` (meta + totals + checkpoints + details, all sequential, each one a separate file open/modify/close round trip). A crash anywhere in that sequence — PC freeze, BSOD, mid-finalize close, antivirus quarantine grabbing the file — left a half-written run on disk: `[meta]` complete, `[totals]` missing, `[details]` count out of sync with the actual rows. The new implementation builds the full INI in memory and writes it through `AtomicWriter` (`.tmp` + `FileMove`), so the destination only ever sees a fully-formed file or no change at all. Same byte-compatible UTF-16 LE BOM output that `IniWrite` used to produce, so existing runs on disk and the `Load` path stay unchanged. The pure `_SerializeBuildResultToIni(buildResult, currentRunId, totalMs)` helper that builds the content is testable in isolation, and Save now also routes its failure path through the injected `WarningSink` instead of returning `true` regardless of what happened on disk. Two new tests cover the atomicity contract: no `.tmp` orphan after a successful save, and `Save` returns `false` + warns on writer failure.

- **Hydration ordering bug (RunStatsRecorder silently dropped hydrated runs).** `SpeedKalandraApp.__New` used to call `RunService.Hydrate` mid-construction. When the loaded state had an active run, `Hydrate` published `Evt.RunStarted{hydrated:true}` before `RunStatsRecorder` (and other downstream subscribers) had been created. Finalizing the hydrated run later produced a snapshot with `runId=""`, which `RunHistoryRepository.Save` silently rejected — the run was lost. `Hydrate` is now deferred to the end of `__New` after `_WireEventHandlers()`, and `ZoneTrackingService._OnRunStarted` respects the `hydrated:true` flag so the in-flight event no longer wipes the totals that were just restored from disk. Regression tests added in `zone_tracking_service_tests.ahk` and `speedkalandra_app_integration_tests.ahk`.
- **`LoadingTotalsService` wiped hydrated `_totalMs` on `RunStarted{hydrated:true}`.** Sibling of the hydration-ordering bug above: the composition root hydrates the loading total from disk via `loadingTotals.Hydrate(loadingMs)`, but `_handlerRunStarted` was hard-wired to `Reset()` and so the deferred `RunService.Hydrate()` event zeroed the value immediately after restoration. Loading time of an in-progress run was lost on every reload. Handler now goes through `_OnRunStarted(data)` which honours the `hydrated:true` flag, mirroring `ZoneTrackingService`. Regression tests `run_started_with_hydrated_flag_preserves_total_ms` and `run_started_without_hydrated_flag_wipes_total_ms` added in `loading_totals_service_tests.ahk`.
- **`LoadingDetectionService` false positives from non-game windows.** The default window provider matched the substring `"Path of Exile 2"`, which also matched Chrome tabs on the PoE2 wiki and Discord channels with that text. The HUD scanner then sampled pixels from the wrong window. Replaced with an `ahk_exe` scan over the same canonical list `FocusAutoPauseService` uses (`PathOfExile2Steam.exe`, `PathOfExile2_x64.exe`, etc.), and lock every follow-up `WinGetMinMax` / `WinGetPos` to the resolved HWND via `ahk_id`.
- **`UndoLastSave` left personal bests pointing at the deleted run.** Inconsistent with `RunHistoryDialog.Delete`, which already rebuilt PBs. `UndoLastSave` now invokes a new `_RebuildPbsFromHistory` helper after a successful delete (mirrors the dialog), so deleting the most-recent run via the tray immediately purges any PBs the deleted run contributed to.

### Changed

- **Run snapshot save moved off the bus and onto `RunService` pre-publish hooks (`SetOnBeforeFinalize` / `SetOnBeforeCancel`).** The earlier wiring subscribed `RunSnapshotSaver.Save` to `Evt.RunCompleted` and `Evt.RunCancelled` and relied on the bus's FIFO Subscribe order in `SpeedKalandraApp.__New` to run Save before `ZoneTrackingService._OnRunEnded` and `RunStatsRecorder.Reset()` cleared the in-memory state. The contract was implicit, undocumented in the test suite, and easy to break by reordering constructors. `RunService.FinalizeRun()` / `CancelRun()` now invoke a settable callback after mutating timer/state and before publishing the lifecycle event — the composition root wires Save through `SetOnBeforeFinalize(() => snapshotSaver.Save("completed"))` and `SetOnBeforeCancel(() => snapshotSaver.Save("cancelled"))` at the end of `__New`. Subscription order on the bus is now irrelevant for the save flow. A throw inside the hook is caught and logged via the injected `LogService` (new optional 5th constructor parameter on `RunService`, defaults to `NullLogger`), so a misbehaving save can't block widgets and state-clearers from reacting to the lifecycle event. 11 new unit tests in `run_service_tests.ahk` (constructor with explicit log, setter type-checks, hook invocation order, hook-skip when idle, publish-survives-throw, warn on hook throw) plus 1 end-to-end integration guardrail (`cancelled_long_run_saves_to_history_with_zone_totals_intact`) that exercises the previously-uncovered code path: cancel a 5-minute run, assert the history file is written and contains the zone totals. Closes the GSG §14 anti-regression item "FIFO da bus como sutil dependência de ordem de wiring".

- **README and `KNOWN_ISSUES.md` corrected on two long-standing inaccuracies.** (1) The disclaimer claimed SpeedKalandra "does not send inputs", but `SpeedKalandraOnExitHandler` in `speedkalandra.ahk` issues `Send "{Blind}{Ctrl up}{Alt up}{Shift up}{LWin up}{RWin up}"` on script exit — a defensive keyup to release stuck modifiers if AHK terminates while a hotkey is held. The disclaimer now mentions it explicitly and clarifies that it's OS-level housekeeping, not directed at the game. (2) Both README's "Game language" section and `KNOWN_ISSUES.md` claimed zone detection, deaths, level changes and loading detection "work regardless of client language" — but the PoE2 `Client.txt` is translated together with the UI, and every parser in `LogMonitorService` matches English fragments (`[SCENE] Set Source`, `has been slain`, `is now level`, `Generating level N area X with seed`, `[WINDOW] Lost focus`). On a non-English client the parsers never fire. Both files now state clearly that SpeedKalandra targets the English client; only loading detection (pixel-based) is genuinely language-agnostic, and only auto-start/auto-finalize have user-configurable regex.

- **Stripped historical version/phase/bug tags from source comments (first pass).** Comments in `src_v2/core/log_service.ahk`, `src_v2/core/event_bus.ahk`, `src_v2/app/bus/events.ahk` and `src_v2/app/services/log_monitor_service.ahk` carried tags like `v17.15 (Bug #32)`, `(v0.1.4)`, `(Wave 3)`, `(Phase 9.2)`, `(R7)` and `(Fase 1)` that documented *when* a change happened during development. Those tags add noise without explaining anything actionable about the code today — the bug history lives in this file and in `git log`, so the source can keep just the technical "why". All 16 occurrences across the four files were reworded in place: the technical explanation stays, the tag disappears. No behaviour change, no test change (suite still 1721 PASS). Other files (services, repositories, UI) were spot-checked and look clean; if residue surfaces in a later read a second pass can sweep them.

- **`EventTraceLogger` is now opt-in.** New `[Diagnostics].EventTracingEnabled` INI flag (default `0`). When false, the bus interceptor is constructed but never registered, so a normal install never persists raw `Client.txt` lines into `speedkalandra.log`. Users who need event-level traces for a bug report flip the flag (see `CONTRIBUTING.md`). Documented in `AppSettings` and surfaced via `SettingsRepository._{Load,Save}Diagnostics`.
- **Silent `try` sweep in critical paths.** Failures in settings save, run-data flush (5 s tick + shutdown), PB update, act-checkpoint capture, hotkey rebind, log-monitor restart, hydrated-run discard/finalize, zone-totals clear, death penalty application, and disclaimer/setup persistence are now logged via `LogService.Warn` instead of being swallowed by bare `try`. Same treatment at the infrastructure layer (`RunStateRepository.SaveZoneTotals`, `RunHistoryRepository._EnsureDir`) via `OutputDebug`, mirroring the existing convention in `RunHistoryRepository.Delete`. Legitimate silent `try` in lifecycle teardown (`Stop`, `Dispose`, `Hide`) and cosmetic side effects (`TrayTip`, `log.Info`) are kept intentionally.
- **Boot-time modals extracted from `SpeedKalandraApp` into a new `BootPrompts` class.** The disclaimer dialog, the Client.txt setup dialog, and the hydrated-run resume/finalize/discard prompt — together ~360 lines — used to live as private methods on the composition root. They are now a thin coordinator class owning no state of its own, constructed in `__New` with a `persistFn` callback and references to the services it needs (`logMonitor`, `runService`, `timer`, `log`). `Start()` calls three public methods (`ShowDisclaimerIfNeeded`, `PromptLogFileSetupIfNeeded`, `PromptHydratedRun`) in sequence. Pure refactor — no behavior change. 10 unit tests cover the guard clauses (headless, already-acknowledged, valid existing path, missing/inactive run service). `src_v2/app/app.ahk` is now ~360 lines shorter as a result.
- **Run finalization + undo extracted into `RunSnapshotSaver`.** `_SaveRunSnapshot`, `UndoLastSave`, `_MarkUndoableSave`, `_RebuildPbsFromHistory`, `_ExpireUndoableSave` (and the `MIN_CANCELLED_SAVE_MS` constant, renamed to `MIN_SAVE_MS` to match its actual semantics) moved out of `SpeedKalandraApp` together with the `_lastSavedRunId` / `_undoTimerFn` state they own. The `RunCompleted` / `RunCancelled` subscriptions in `__New` now route through a late-bound callback (`(data) => this._snapshotSaver.Save(...)`) — the saver itself is constructed near the end of `__New` once every collaborator it depends on exists. `app.UndoLastSave()` remains as a one-line delegate so the tray callback in `speedkalandra.ahk` is unchanged. Pure refactor — no behavior change. 10 unit tests cover the threshold gate, undo state machine, and PB rebuild semantics.
- **Persistence flow extracted into `RunStatePersister`.** `_PersistRunData` (5 s tick), `_PersistRunDataFull` (final flush from `Stop()`), `_PersistSettings`, and `_ComputeTotalsHash` moved out of `SpeedKalandraApp`, together with the dirty-cache fields `_lastSavedLoadingTotal` / `_lastSavedZoneTotalsHash` that they own. `Start()` now schedules `() => this._persister.Tick()` instead of `() => this._PersistRunData()`. The `_persistFn` closure passed to widgets, dialogs, and `BootPrompts` routes through `this._persister.PersistSettings()` so every settings write goes through the same instance. `__New` primes the cache via `PrimeLoadingTotalCache` / `PrimeZoneTotalsCache` after hydration so the first tick doesn't redundantly rewrite the just-loaded state. Pure refactor — no behavior change. ~30 unit tests cover skip-cache, full-flush, cache priming, and reset semantics.
- **Hot-reload and destructive-action handlers extracted into `LiveReconfigurationHandlers`.** Three handlers — `_OnDeathApplyTimerPenalty` (timer penalty applied on `Evt.DeathDetected`), `_OnHotkeysChanged` (Stop + Hydrate + Start on the hotkey service), and `_OnResetPersonalBestsRequested` (confirmation dialog + reset) — moved out of `SpeedKalandraApp` into a small class constructed in `__New` with the collaborators each handler needs (`cfg`, `timer`, `hotkeyService`, `personalBest`, `log`). The subscriptions in `_WireEventHandlers` are now one-line delegates: `(data) => this._reconfig.ApplyDeathPenaltyToTimer(data)` etc. `_OnLogFilePathChanged` stays inline because it mutates `_logMonitorTimer` on the composition root and the extra plumbing isn't worth the size win. Pure refactor — no behavior change. 16 unit tests cover the guard clauses, payload-vs-cfg fallback, and the headless reset path.

### Build

- **`build-dist.ps1` runs the AHK test suite as a release gate.** A red tree no longer produces a distribution. Bypass with `-SkipTests` (only for local iteration; recursive call from `tests_v2/build/test_build_dist.ps1` also uses this flag).
- **`build-dist.ps1 -Zip` now writes a SHA256 sidecar.** The `.zip` is accompanied by `.zip.sha256.txt` in the coreutils `sha256sum` format (`<hex>  <filename>`). Verifiable on Linux/macOS with `sha256sum -c` or on Windows with `Get-FileHash`.
- **`tests_v2/build/test_build_dist.ps1` added.** PowerShell test that builds a fake project with personal-data sentinels (`speedkalandra.ini`, `Client.txt`, `data/runs/*.ini`, `debug/`, `BKP/`, `_LIXEIRA/`, dev-only markdown, etc.) under `%TEMP%`, runs `build-dist.ps1 -Zip -SkipTests`, then asserts (a) every sentinel is absent from both the staged dir and the produced `.zip`, (b) the must-keep files (`speedkalandra.ahk`, `data/zones.csv`) are present, (c) the SHA256 sidecar exists, parses, and matches the .zip. Designed to live alongside the AHK suite without being driven by it — the AHK runner is the wrong tool for testing PowerShell build behaviour.
- **`build-dist.ps1` rejects descendant destinations.** Running `.\build-dist.ps1 -DestDir ".\dist"` used to enter a recursive copy. Now both ancestor and descendant relationships between source and destination are rejected with a clear error.
- **CI reintroduced.** New `.github/workflows/test.yml` runs the `tests_v2/` suite on `windows-latest` for every push and PR; installs AutoHotkey v2 via Chocolatey, runs in headless mode (`SPEEDKALANDRA_TEST_NO_GUI=1`), and uploads `tests_output.log` as an artifact on failure. The previous attempt had been disabled because `#Warn All, MsgBox` in `run_tests.ahk` made the runner exit with a non-zero code even on all-green runs — routed to `OutputDebug` instead.

### Fixed

- **PB persistence failures no longer disguised as success.** Five mutations in `PersonalBestService` (`UpdateFromRun`, `Reset`, `LoadFromExternal`, `SetAsRunPb`, `RebuildFromHistory`) used to end with `try this._PersistToRepo()` — a bare silent try that swallowed both repository-returned-false (disk full, locked file, encoding error) and actual exceptions. The PB landed in memory but never reached `data/personal_bests.ini`, and the user saw nothing in `speedkalandra.log`. They now route through a new `_TryPersistOrWarn(context)` helper that surfaces both branches as visible `[PB]` WARNs while preserving the original behaviour of NOT breaking the finalize flow on persist failure. `_PersistToRepo` returns the bool from `_repo.Save` so the helper can distinguish "repo refused" from "repo threw".
- **`RunStateRepository.LoadZoneTotals` graded failure handling.** Used to silently return an empty Map on any `FileRead` failure, masking lock / permission / encoding errors that lose run-in-progress zone time. Now follows the graded taxonomy documented in `ARCHITECTURE.md` § 14: file missing is silent (expected new state); `FileRead` throws produces a `[RunState]` WARN with the exception; individual malformed lines are skipped silently (resilient to manual edits); a non-empty file where every line failed to parse produces a corrupt-file WARN.

### Changed

- **`WarningSink` introduced in `core/` for infra/services without a LogService dependency.** Repositories were logging Save / Delete / EnsureDir failures via `OutputDebug` only, which is invisible to the user without DbgView. The new `WarningSink` is a duck-typed observability channel (three implementations: `NullWarningSink`, `LogServiceWarningSink`, `InMemoryWarningSink`) injected into `PersonalBestRepository`, `RunStateRepository`, `RunHistoryRepository`, and `PersonalBestService` at composition-root construction. Each receives a tagged sink (`"PB"`, `"RunState"`, `"RunHistory"`) so a grep of `[PB]` / `[RunState]` / `[RunHistory]` in `speedkalandra.log` isolates failures by source layer. All `OutputDebug("X.Y failed: ...")` calls in those files were replaced with `this._warn.Warn(...)`. Default parameter is `""` so existing tests that construct repos with the legacy positional signature continue to pass (degrade to `NullWarningSink`). 41 new tests cover the sink contract, the persist-failure paths in `PersonalBestService` (with stub repos that return false / throw), and the graded `LoadZoneTotals` taxonomy.

### Documentation

- **`SECURITY.md` rewritten with explicit scope.** Replaces the minimal stub with a project-appropriate threat model (local-first, no network, no telemetry), enumerated in-scope categories (unexpected network activity, private data leakage, file writes outside the project directory, release packages containing personal files, malicious behaviour in distributed artifacts) and out-of-scope categories (incorrect timing, game-log format changes, AV false positives without reproducible unsafe behaviour, anything in forks). Disclosure preferences and the private channel are kept.
- **Issue templates added.** `.github/ISSUE_TEMPLATE/bug_report.md` and `.github/ISSUE_TEMPLATE/feature_request.md` plus a `config.yml` that disables blank issues and points security and known-limitations cases at the right docs. Bug template requests SpeedKalandra version, AHK version, Windows version, PoE2 client language, run state at the time, `EventTracingEnabled` flag, reproduction steps, expected vs actual, log snippet (with a privacy-review note), screenshot, and reproducibility category.

- **Error handling policy documented.** New § 14 in `ARCHITECTURE.md` formalizes the rule the codebase already follows for when `try` / `catch` should log on failure: *log where silence would hide data loss, broken state, or actionable failure*. Lists the four buckets every catch in the project falls into (logged via `LogService.Warn`; silent during lifecycle teardown; silent for cosmetic side effects; silent UI fallbacks that abort safely), with concrete examples and a decision tree for new code. Cross-referenced from § 13 (Conventions) and from `CONTRIBUTING.md`. One borderline silent catch found during the audit — `RunSnapshotSaver` reading the act-checkpoints map for the saved-run payload — was promoted to bucket (1) with an explicit `Warn` because a failure there would put an empty checkpoints map into the on-disk run file with no signal (data loss in disguise).

- **Historical version tags scrubbed from source comments.** Roughly 40 files across `src_v2/` and `tests_v2/` carried internal-timeline markers in their headers and inline comments — `Wave N`, `Phase A1` / `Phase B2.1` / `Phase 6 Option A` / `Phase 9.2`, `v17.x` and `v0.1.x` version tags, `Bug #N`, `POST-DEMOLITION VERSION:`, `(Item N)`, `(refactor R7)`, `(R11.1)` — all of which described how the code reached its current shape rather than what the code does today. Bug IDs in test names (`bug9_*`, `bug21_*`, `W4.1`, `W5.1`, `W9.1`, `W9.2`) were intentionally kept — they are stable contracts with `tests_v2/REGRESSION-COVERAGE.md`.
- **Disclaimer rewritten and made canonical across all three surfaces.** The README, the in-app boot modal (`BootPrompts.ShowDisclaimerIfNeeded`), and `CONTRIBUTING.md` previously carried three different versions, several of them with self-deprecating phrasing about being a player rather than a developer, promises of slow support, and an explicit "I won't pretend I wrote the architecture" line. A single shorter version now lives in the README and is mirrored faithfully in the boot modal. The substantive points (AI-assisted development, anti-cheat / TOS posture, use-at-your-own-risk, GPL forks) are preserved; the apologetic framing is gone.
- **Hard-coded test counts replaced by stable phrasing.** README, `CONTRIBUTING.md`, `tests_v2/README.md`, and `ARCHITECTURE.md` previously stated `~1569` (and the per-layer breakdowns in two of those files disagreed numerically). ## [v0.1.3] — 2026-05

### Added

- **Client.txt setup dialog on first boot.** When `cfg.logFile` is empty or points to a missing file, a modal dialog appears with a pre-filled Steam path and a Browse button. Cancel calls `ExitApp()` — the app refuses to run without a valid log path.
- **Death penalty applied to the live timer.** `TimerService.AddPenaltyMs(ms)` and an `_OnDeathApplyTimerPenalty` handler subscribed to `Evt.DeathDetected`. With `cfg.deathPenaltyEnabled = true` (default) the timer jumps forward by `cfg.deathPenaltyMs` (default 2 min 30 s) the moment a death is detected, so the user no longer sees an inconsistency between the overlay timer and the post-finalize plot.

### Changed

- **Settings UI cleanup.** Removed the unused `Patch` field; the value is still stored internally (`cfg.gamePatch = "Unknown"`) for back-compatibility with old saved runs. Fixed a visual bug where the Client.txt Edit auto-expanded vertically when the path was long.

### Tests

- 19 new tests (13 unit covering `TimerService.AddPenaltyMs`, 6 integration covering the death-penalty handler's guard paths).

## [v0.1.2]

### Fixed

- **Loading-detection timeouts no longer silently discarded** (Bug #5). When a loading screen exceeded `cfg.loadingVisualMaxMs` (default 90 s, slow PCs), `LoadingDetectionService` correctly detected the timeout but then dropped the event via a `durationMs > maxMs` filter in `_End`. The HUD-return event for that loading never reached the bus, so the run plot underestimated total loading time. Timeouts now publish with the real duration (no clamp).

### Changed

- **Duration formatting consolidated** (audit #19). Four near-identical copies of `FormatMs` were collapsed into a single `Duration.FormatMs` in `domain/values/duration.ahk`. All call sites updated.
- **Multi-line log entries are quieter** (audit #26). Stack-trace style entries now indent continuation lines instead of producing N separate timestamped rows.

## [v0.1.1]

### Fixed

- **`TextEncoding.ConvertUtf16ToUtf8` and `MigrateIniToUtf8` removed** (Bug #2). The migration produced INIs with a UTF-8 BOM, and AHK v2's `IniRead(path, section, key, default)` silently returns the default on UTF-8 BOM files. Result: every key-based read after migration returned the fallback value — settings, run state, personal bests all looked freshly-installed. Reverted to UTF-16 LE BOM throughout. Documented in `text_encoding.ahk` and `ARCHITECTURE.md` §14.
- **"Lechtansi" bug: zone timer kept ticking during pause.** A `[SCENE]` line emitted by the game while alt-tabbed became a `ZoneChanged` event, which restarted `_startMs` in `ZoneTrackingService` even though `TimerPaused` had been observed earlier. Added a `_timerPaused` flag so `_OnZoneChanged` respects pause state.
